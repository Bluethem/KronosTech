# Stack Tecnológico

## Backend: Rust + Axum

El backend del sistema se desarrollará utilizando **Rust**, un lenguaje moderno, seguro y de alto rendimiento.
Rust destaca especialmente en aplicaciones donde se necesita:

* Máxima eficiencia y bajo consumo de recursos
* Concurrencia segura sin *data races*
* Fiabilidad en entornos de alta carga

<div align="center">
  <img src="/documentacion/assets/images/3/rust-logo.svg" alt="Logo Rust" width="200"/>
</div>

### Ventajas del backend en Rust:

* **Seguridad en memoria garantizada** sin necesidad de garbage collector.
* **Alto rendimiento**, comparable al de C/C++.
* Ideal para servicios backend de misión crítica y microservicios de alta concurrencia.
* Compilación estática → distribuciones más fáciles y seguras.

### Framework: Axum

El framework elegido es **Axum**, uno de los más modernos y robustos en el ecosistema Rust.

Sus principales beneficios:

* Basado en **tokio**, el runtime asíncrono más eficiente del ecosistema Rust.
* Enrutamiento declarativo y soporte nativo para *extractors*, *middlewares* y Tower.
* Fácil integración con bases de datos mediante **SQLx** o **SeaORM**.
* Ideal para APIs REST de alto rendimiento.

## Frontend: Svelte + SvelteKit + TypeScript

El frontend se desarrollará con **Svelte**, un framework que destaca por su rendimiento y simplicidad, ya que compila a JavaScript optimizado sin un runtime pesado.

<div align="center">
  <img src="/documentacion/assets/images/3/svelte_Logo.svg" alt="Logo Svelte" width="200"/>
</div>

### Ventajas de Svelte:

* Componentes altamente reactivos sin Virtual DOM.
* Paquetes mucho más livianos que React o Vue.
* Código más simple y limpio, ideal para equipos pequeños o medianos.

### SvelteKit

SvelteKit es el meta-framework oficial para construir aplicaciones completas.
Ofrece:

* **SSR (Server-Side Rendering)**
* **Rutas basadas en archivos**
* Manejo de formularios con *actions*
* Carga optimizada de datos
* Integración rápida con APIs externas (como el backend Axum)

### TypeScript

El proyecto utilizará **TypeScript** tanto en el frontend como en algunos modelos compartidos.

Beneficios:

* Autocompletado y tipado estricto
* Menos errores en producción
* Código más fácil de mantener

## Base de Datos: PostgreSQL

El sistema utilizará **PostgreSQL**, una base de datos relacional altamente confiable, escalable y robusta.

<div align="center">
  <img src="/documentacion/assets/images/3/Postgresql_elephant.svg" alt="Logo PostgreSQL" width="200"/>
</div>

### Ventajas de PostgreSQL:

* Cumple estrictamente con **ACID**, garantizando integridad de datos.
* Alta performance incluso con millones de registros.
* Soporte híbrido SQL + JSONB (ideal para datos semiestructurados).
* Funciones avanzadas:

  * Full Text Search
  * Particiones
  * Extensiones como PostGIS
* Integración nativa con SQLx, Prisma Client (Rust), y ORMs del ecosistema.
